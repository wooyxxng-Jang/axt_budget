# AXT 예산 집행 자동화 (axt_budget)

매월 세인트에서 다운로드하는 예산집행상세 파일(.xls, 실제로는 HTML 테이블)을 업로드하면
아텍(ATC) 사용분을 자동 분리하고, 전표일자 기준 월별·비목별로 집계하여 배정예산 대비
잔액을 자동 계산하는 Google Sheets + Apps Script 시스템.

스펙 문서: `AXT_예산자동화_스펙.md` 참고 (Phase 1 구현)

## 구성

```
.clasp.json          # clasp 프로젝트 설정 (scriptId는 최초 연결 시 입력)
src/
  appsscript.json    # Apps Script 매니페스트
  Config.js          # 시트/컬럼 상수, 공통 유틸 (upsert 키, 금액/일자 파싱)
  HtmlParser.js      # 세인트 .xls(HTML) 파서
  Classifier.js      # 부서 귀속 판정 (프로젝트prefix → 강한키워드 → 교수성함 → 타학과 → 검토)
  DetailRouter.js    # 상세분류(세부라인) 자동 라우팅
  Master.js          # 배정예산 마스터(외부 구글시트) 참조
  Ledger.js          # 원장 upsert (키: 참조전표+개별항목)
  Upload.js          # 업로드 파이프라인 (스펙 7장 처리 흐름)
  ReviewInbox.js     # 검토대기함
  ScholarshipInbox.js# 장학금 분류 대기함 (전표유형=등록금)
  DetailQueue.js     # 상세분류 확인 대기열
  Summary.js         # 월별집계 + 잔액 재계산
  Drilldown.js       # 월별집계 셀 → 원장 드릴다운 모달
  AuditLog.js        # 감사로그 (자동/수동 제외 건)
  SetupSheets.js     # 시트 초기화/점검 (7개 시트 + 시드 데이터)
  Main.js            # 메뉴/사이드바 진입점
  Sidebar.html       # 키워드 관리 사이드바 UI
  UploadDialog.html  # 파일 업로드 다이얼로그
```

## 시트 구성 (초기화 시 자동 생성)

| 시트 | 역할 |
|---|---|
| 월별집계 | 자금/비목/상세분류 × 월별 합산 + 배정액 대비 잔액 (매번 원장 기준 재계산) |
| 원장 | 아텍 귀속 확정 전건 누적. upsert 키 = 참조전표+개별항목 |
| 검토대기함 | 판정 불가 건. `판정` 컬럼에 포함/제외 선택 → 메뉴로 반영 |
| 상세분류 확인 대기열 | 상세분류 자동 라우팅 실패 건. `선택상세분류` 드롭다운 선택 → 메뉴로 반영 |
| 장학금 분류 대기함 | 등록금 유형 전표 신규 건. `귀속`(아텍/타학과) + 상세분류 태깅 → 메뉴로 반영 |
| 설정 | A~B: 키워드 / D~H: 상세분류 라우팅 규칙 / J~K: 일반 설정(마스터 시트 ID 등) |
| 감사로그 | 자동/수동 제외된 건 전체 기록 (스팟체크용) |

## 최초 1회 설정

1. **Node.js 설치** (LTS) → `npm install` (로컬에 clasp 설치됨)
2. **clasp 로그인** — 전공사무실 구글 계정으로:
   ```
   npx clasp login
   ```
   (브라우저 OAuth 인증. 생성되는 `~/.clasprc.json`은 절대 커밋 금지 — .gitignore에 포함됨)
3. **Apps Script API 활성화**: https://script.google.com/home/usersettings 에서 ON
4. **스프레드시트 연결**:
   - 운영용 구글 시트를 새로 만들거나 기존 시트 열기
   - 확장 프로그램 > Apps Script > 프로젝트 설정에서 **스크립트 ID** 복사
   - `.clasp.json`의 `scriptId`에 붙여넣기
5. **배포**:
   ```
   npx clasp push
   ```
6. 시트 새로고침 → 메뉴 **[AXT 예산 > 시트 초기화/점검]** 실행
7. 설정 시트에서:
   - J~K열: `MASTER_SPREADSHEET_ID`(배정예산 마스터 시트 ID), `MASTER_SHEET_NAME` 입력
   - D~H열: 라우팅 규칙의 `(마스터 상세분류명으로 교체)`를 실제 상세분류명으로 교체

## 매월 운영 흐름

1. 세인트에서 회계연도 전체 기간으로 예산집행상세 다운로드
2. 메뉴 **[AXT 예산 > 세인트 파일 업로드]** → 파일 선택
3. 자동 판정 결과 확인 (99%+ 자동 처리)
4. 남은 건 처리:
   - 검토대기함: `판정` 선택 → **[검토대기함 → 확정 반영]**
   - 장학금 분류 대기함: `귀속`(+상세분류) 태깅 → **[장학금 태깅 → 반영]**
   - 상세분류 확인 대기열: `선택상세분류` 선택 → **[상세분류 선택 → 반영]**
5. 월별집계에서 확인. 월 금액 셀 선택 후 **[선택 셀 드릴다운]** 으로 구성 내역 확인
6. 키워드 보강: **[키워드 관리 (사이드바)]** 또는 설정 시트 직접 편집

한번 확정된 건은 upsert 키 기준으로 영구 저장되어, 재업로드해도 다시 묻지 않는다.

## 개발 워크플로

```
코드 수정 → npx clasp push → 시트에서 확인 → git commit → git push (GitHub 백업)
```

- 마스터 배정예산 시트는 읽기 전용으로 참조만 함 (실적/잔액은 이 시트의 월별집계에서 관리)
- `appsscript.json` 스코프: spreadsheets(마스터 참조용), script.container.ui(메뉴/사이드바)
