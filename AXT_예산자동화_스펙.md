# AXT 예산 집행 자동화 시스템 — 개발 스펙 (Final)

## 0. 목적
매월 세인트에서 다운로드하는 예산집행상세 파일(.xls, 실제로는 HTML 테이블)을 업로드하면,
- 지융미 행정실 사용분과 아텍 사용분을 자동 분리하고
- 전표일자 기준 월별·비목별로 배정예산 마스터에 자동 합산하고
- 재업로드해도 기존 누적 데이터가 깨지지 않게

만드는 것이 목표. Phase 2(업무협조 파일 자동 생성)는 별도 확장.

## 1. 기술 스택 — Google Sheets + Apps Script

| 비교 | Python exe | 웹 호스팅 | **Sheets + Apps Script** |
|---|---|---|---|
| 배포/업데이트 | 재빌드 필요 | 용이 | 스크립트 저장 즉시 반영 |
| 보안 | 로컬은 안전 | 외부 호스팅 부담 | 학교 구글 계정 권한 그대로 |
| 후임자 인계 | exe+소스 별도 관리 | 배포 인프라 인계 필요 | 시트 공유 = 인계 끝 |

- 개발 도구: **clasp**(Apps Script CLI)로 Claude Code에서 로컬 개발 → `clasp push`로 배포
- 진행 중인 챗봇·대여 시스템도 Google Sheets 기반이라 운영 지식이 한 곳에 모임
- 실행 범위: Apps Script는 구글 서버에서 실행되지만, 전공사무실 구글 계정으로 바인딩된 이 스프레드시트에서만 동작 → 사실상 이 계정을 쓰는 사람만 실행 가능. 외부 서버 호스팅·공개 배포 없음

## 2. 입력 데이터

**세인트 다운로드 파일**
- 확장자는 .xls이나 실제로는 HTML 테이블, UTF-8 인코딩
- 매번 **회계연도 전체 기간**으로 다운로드 (부분 기간 아님)
- 핵심 컬럼: `수입/지출`, `자금`, `자금명`, `약정항목`, `약정항목 명`, `전표상태`, `전표유형명`, `참조전표`, `개별항목`, `전표일자`, `사용금액`, `텍스트`, `지급일자`
- **Upsert 키: `참조전표` + `개별항목`** (전수 검증 결과 중복 없음, 안정적인 고유키)

**배정예산 마스터 (기존 구글시트)**
- 자금코드/약정항목코드/상세분류/배정액이 이미 존재
- 이 시트를 그대로 참조 — 별도 재입력 없이 여기에 실적/잔액 컬럼을 붙이는 구조로 설계

## 3. 부서 귀속 판정 로직 — 키워드 우선순위

부서 공용 prefix(과목/사업 코드)는 여러 학과가 함께 쓰기 때문에 그 자체로는 부서를 구분하는 신뢰할 수 있는 신호가 아니다. **텍스트 내용에 포함된 키워드**를 1차 판정 기준으로 삼는다.

**판정 순서 (위에서부터 순차 적용, 먼저 매치되는 규칙 우선)**

1. **프로젝트 전용 prefix** (아텍만 쓰는 고유 프로그램명): `Kakao테크포임팩트수업`, `ArtWithImpact` → **자동 포함**
2. **강한 확정 키워드**: `아텍`, `ATC` → **자동 포함**
3. **아텍 교수님 성함 포함 + 타학과 키워드 미포함** → **자동 포함**
   - 교수님 성함 목록(설정 시트에서 관리): `최용순`, `김주섭`, `김상용`, `정다샘`, `정해동`, `오준호`, `이선옥`, `주진호`
4. **타학과 키워드 포함 + 교수님 성함 미포함** → **자동 제외**
   - 타학과 키워드(설정 시트에서 관리): `캡스톤`, `지융미`, `신방`(신문방송학과), `미엔`(미디어엔터테인먼트), `미컴원`
5. **성함과 타학과 키워드가 동시에 있거나, 둘 다 없는 경우** → **검토대기함**

**실데이터 검증 결과 (회계연도 401건 기준)**

| 분류 | 건수 | 금액 |
|---|---|---|
| 자동포함(프로젝트prefix) | 17 | 1,223,800원 |
| 자동포함(강한키워드: 아텍/ATC) | 67 | 12,792,443원 |
| 자동포함(교수성함) | 8 | 1,220,000원 |
| 자동제외(타학과키워드) | 232 | 17,070,175원 |
| 장학금 대기(등록금 유형, 별도 워크플로) | 76 | 22,518,800원 |
| 검토필요 | 1 | 66,000원 |

전체의 99.75%가 자동 판정되며, 나머지는 검토대기함에서 원클릭으로 확정한다.

- prefix는 포함/제외 판정에는 쓰지 않되, 하나의 자금+비목 코드 안에 상세분류(세부 라인)가 여럿일 때 이를 구분하는 보조 신호로는 사용한다 (4장 참고)
- 키워드 리스트(강한 확정 키워드/교수님 성함/타학과 키워드)는 설정 시트에서 관리하며, 사이드바 UI로 그때그때 추가·수정·삭제 가능

### 3-1. 키워드 관리 UI (사이드바)
- Apps Script HTML Service로 **간단한 사이드바** 제공: 키워드 추가/수정/삭제를 버튼 클릭으로 처리
- 입력 즉시 설정 시트에 반영, **코드 재배포 불필요**
- 중복 입력 방지, 빈 값 검증 등 기본적인 오류 방지 로직 포함
- 설정 시트를 직접 열어 셀 편집도 가능(사이드바는 편의 기능이지 유일한 경로는 아님)

### 3-2. 검토대기함 처리
- 위 규칙에 다 해당 안 되는 건(성함·타학과 키워드 둘 다 없음, 또는 둘 다 있어 충돌) → **검토대기함**에 기록
- 사람이 포함/제외 확정 → 확정 즉시 원장에 반영, upsert 키로 영구 저장(다음달 재확인 불필요)
- 확정 이력은 향후 키워드 규칙 보강의 참고 자료로 활용

## 4. 상세분류(세부 라인) 매핑

배정예산 마스터는 **자금코드+약정항목코드+약정항목명** 조합 32개 그룹 중 26개는 상세분류가 1개뿐이라, 비목 레벨 집계가 곧 상세분류 레벨 집계가 되어 100% 자동이다.

**상세분류가 2개 이상인 6개 그룹은 두 갈래로 나뉜다:**

**A. 장학금 분류 대기함 워크플로에 자연스럽게 흡수되는 경우**
- `FGC9361000(43221040) 교내장학금-기금`(상세분류 4개), `TBC0183(43221030) 교내장학금-근로`(상세분류 2개)
- 실거래 텍스트가 전부 `장학금` 형식(전표유형=등록금)이라 어차피 사람이 태깅하는 항목
- 태깅 UI에서 "아텍/타학과" 선택 외에 **"상세분류(세부항목) 선택"** 드롭다운을 추가로 둔다

**B. 자동 포함까지는 확정되나, 상세분류 하위 라우팅이 필요한 경우**
- `FGC9361000(42381110) 회의비`: `Kakao테크포임팩트수업` vs `아텍/ArtWithImpact` prefix로 자동 라우팅
- `TBC0184(42371110) 홍보비`: 실거래 전부 특정 서브라인에 해당하여 자동 라우팅
- `TCC0185(43251010) 학생활동지원비`: 자동 라우팅 실패 시 **"상세분류 확인 필요" 대기열**로 보내고, 후보 상세분류 목록을 보여줘서 원클릭 선택

**설계 원칙**: 비목(자금+코드) 레벨 합산·잔액은 100% 자동. 상세분류(세부 라인) 레벨은 "자동 매칭 우선 + 실패 시 사람이 원클릭으로 확정" 구조. 실사용 내역이 예산 기획 문구와 정확히 일치하지 않는 사례가 있어 완전 자동은 약속하지 않는다.

## 5. 장학금(전표유형=등록금) 별도 처리 워크플로
- 이 건들은 prefix가 없고 타 학과 장학금도 섞여 들어옴 → 자동 분류 대상에서 제외
- 별도 시트 **"장학금 분류 대기함"**을 만들어, 신규 건만 사람이 매달 아텍/타학과 여부 + 해당 시 상세분류(세부항목)를 태깅
- 한번 태깅되면 upsert 키 기준으로 영구 저장 → 재업로드(회계연도 전체 기간) 시 이미 태깅된 건은 다시 묻지 않고, 신규 건만 대기함에 노출

## 6. 시트 구성

| 시트명 | 역할 |
|---|---|
| **원장 (Ledger)** | 아텍 귀속 확정된 전건 누적. upsert 키=참조전표+개별항목 |
| **월별집계** | 배정예산 마스터와 동일한 레이아웃(자금/비목/상세분류 × 월별 컬럼), 전표일자 기준 자동 합산 + 잔액 자동 계산 |
| **검토대기함** | 교수성함/타학과 키워드 둘 다 없거나 충돌하는 건. 사람이 포함/제외 결정 |
| **상세분류 확인 대기열** | 아텍 귀속은 확정됐으나 상세분류(세부라인) 자동 라우팅 실패한 건. 후보 상세분류 중 원클릭 선택 |
| **장학금 분류 대기함** | 등록금 유형 전표 중 신규 건. 아텍/타학과 태깅 + 해당 시 상세분류(세부항목)도 선택 |
| **설정 (키워드 관리)** | 강한 확정 키워드/교수님 성함/타학과 키워드 리스트, 운영자가 사이드바 UI 또는 직접 셀 편집으로 수정 가능 |
| **감사로그** | 자동 제외된 건 전체 기록 (스팟체크용) |

- **월별집계 셀 클릭 시 드릴다운**: 해당 월·비목 합산 내역이 원장 어느 행들로 구성됐는지 바로 확인 가능하게 (필터 링크 또는 QUERY 함수 기반 서브뷰)

## 7. 처리 흐름 요약

1. 시트 메뉴 "예산 업로드" 클릭 → 세인트 .xls 파일 업로드
2. 스크립트가 HTML 파싱
3. 전표유형=등록금(장학금) 건 분리 → 장학금 분류 대기함으로 (기존 태깅 건 제외, 신규만)
4. 나머지 건 → 3장 판정 순서(프로젝트prefix → 강한키워드 → 교수성함 → 타학과키워드 → 검토) 적용
   - 자동 포함 확정 → 상세분류 라우팅 단계로 (아래 5번)
   - 자동 제외 확정 → 감사로그
   - 판정 불가(충돌·둘다 없음) → 검토대기함
5. 자동 포함 건 중, 해당 자금+비목코드에 상세분류가 2개 이상인 경우 → 서브라인 키워드/prefix 규칙 적용
   - 매칭 성공 → 원장 upsert (상세분류 확정)
   - 매칭 실패 → 상세분류 확인 대기열 (사람이 후보 중 원클릭 선택 후 원장 반영)
   - 상세분류 1개뿐인 경우 → 바로 원장 upsert
6. 검토대기함·장학금대기함·상세분류확인대기열에서 사람이 확정한 건 → 원장에 반영
7. 원장 데이터를 전표일자 기준 월별집계 시트에 자동 반영, 배정예산 마스터 대비 잔액 갱신

## 10. 확정된 전제
- 전표상태(임시전표/실전표송장/지급수입) 구분 없이 전부 확정 집계
- 세인트 다운로드는 매번 회계연도 전체 기간, 원장은 upsert로 누적
- 배정예산 마스터는 기존 구글시트를 그대로 참조
- 장학금 전표는 자동분류 대상 아님 — 사람이 매달 신규건만 태깅
- 부서 귀속 판정은 텍스트 키워드(교수 성함/아텍·ATC/타학과명) 우선, prefix는 상세분류 라우팅의 보조 신호로만 사용
- 키워드 리스트는 설정 시트 + 사이드바 UI에서 운영자가 직접 그때그때 추가/수정/삭제 가능
- 실행 환경은 전공사무실 구글 계정 하나로 한정 (외부 공유·다중 사용자 불필요)
- 소스코드는 clasp로 관리하여 우영님 개인 GitHub 리포지토리에 백업/버전관리
- 비목(자금+코드) 레벨 집계는 100% 자동, 상세분류(세부라인) 레벨은 자동 매칭 우선 + 실패 시 사람이 원클릭 확정

## 11. 실행 환경 & GitHub 워크플로우

**GitHub 연동 (clasp 기반)**
1. `clasp login` — 전공사무실 구글 계정으로 최초 1회 로그인 (로컬 브라우저에서 OAuth 인증 필요)
2. `clasp clone <scriptId>` — 기존 Apps Script 프로젝트를 로컬로 가져오기 (또는 `clasp create`로 신규 생성)
3. 로컬 디렉토리를 `git init` → 우영님 GitHub 리포지토리에 push
4. **주의**: `.clasprc.json`(인증 토큰)은 절대 커밋 금지 → `.gitignore`에 반드시 추가
5. 이후 개발 흐름: 로컬에서 코드 수정(Claude Code 활용) → `clasp push`로 시트에 반영 → 확인 후 `git commit & push`로 GitHub 백업

**Claude Code 작업 시 참고**
- Claude Code는 `clasp login`의 OAuth 인증 절차는 대신할 수 없으므로, 최초 인증은 우영님이 직접 진행
- 이후 코드 작성·수정·`clasp push`까지는 Claude Code에서 반복 가능
